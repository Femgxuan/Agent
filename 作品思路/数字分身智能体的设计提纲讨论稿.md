# 数字分身智能体的设计提纲讨论稿

## 1. 简介：我们的目标、理论与挑战

### 1.1 最终目标

我们的最终目标，不是构建一个简单的“RAG知识库”或聊天机器人。

我们的目标是为平台上的每一位科研用户（特别是“探究型”专家）构建一个高保真的“**数字分身 (Digital Persona)**”。这个“分身”是一个能7x24小时工作的AI伙伴，它不仅**掌握**用户的知识，更能**模拟**用户的独特思维方式，最终有能力**代表**用户参与“群体智能”讨论班，涌现新的科研思想。

### 1.2 理论基础：什么是真正的“数字分身”？

一个真正的“数字分身”，**绝不仅仅是用户“静态知识”的集合**。

我们的设计理论基础，是认知科学中由著名哲学家迈克尔·波兰尼 (Michael Polanyi) 提出的**“显性知识”与“隐性知识” (Explicit vs. Tacit Knowledge) 理论**。

- **1. 显性知识 (Explicit Knowledge) - “你知道什么”**
  - **定义：** 这是可被言说、被记录的“静态知识”。
  - **对应平台：** `模块一`。即用户撰写或认可的文献、知识库（RAG）。
  - **局限：** 一个只掌握“显性知识”的AI，是一个“博学的图书馆员”，它能**复述**知识，但无法**运用**知识。
- **2. 隐性知识 (Tacit Knowledge) - “你如何思考”**
  - **定义：** 这是我们**真正的目标**。它是用户的科研直觉、心智模型、批判性思维框架——即“动态知识”或“思考过程”。
  - **实例：** 我们通过分析一位用户深度学习10篇文献的完整讨论过程，成功提炼出了其独特的**“CMS（概念-机制-系统）画像”**。这个“画像”就是其“**认知指纹**”（Cognitive Fingerprint）——一个宝贵的“隐性知识”参考。
  - **核心价值：** 这个“认知指纹”是“数字分身”的核心，使其能代表用户参与“群体智能”讨论班或“评审会”，不仅提供知识，更能提供**独一无二的思考框架**（例如，在评审中自动扮演“矛盾检验者”的角色）。

### 1.3 核心挑战

基于“隐性知识”理论，我们的核心挑战变得非常清晰。根据“**波兰尼悖论**”（即“我们知道的远比我们能言说的多”），“隐性知识”无法被直接告知，**它必须通过“观察”用户的行为来捕捉**。

因此，我们的挑战是：

1. **数据保真度：** 如何设计一个流程来“观察”和“捕捉”用户“鲜活的思考过程”？
2. **激励机制：** 如何让用户（尤其是专家）有强大动机，愿意投入时间来让我们“观察”？
3. **用户多样性：** 如何在服务90%“任务型”用户（只想要“显性知识”RAG）的同时，精准“捕获”并激励10%“探究型”专家（他们是“隐性知识”的宝库）？

## 2. UED 设计：低摩擦力、高激励的“三卡片”校准系统

我们的解决方案是：不强迫用户“训练”AI。我们提供一个“**分身进化**”的“游戏化”界面，用户可以按自己的节奏和意愿，自主选择完成三个“校准模块”，以此来解决上述三大挑战。

### 2.1 核心仪表盘：“我的数字分身”

这是用户个性化中心的主界面。UED团队在设计此界面时，必须一目了然地传达**价值**、**状态**和**路径**。

- **引导词：** 仪表盘的引导词必须首先树立“理论大旗”，向用户清晰地传达我们设计的理论基础（显性/隐性知识），以此证明后续“校准模块”的必要性和科学性。

  - *（UED文案示例）*

  > **标题：** 激活您的“数字分身”
  >
  > **正文：** “欢迎来到您的‘数字分身’中心。一个真正的AI科研伙伴，不仅需要学习您的**‘显性知识’**（您撰写的文献和知识库），更需要学习您最宝贵的**‘隐性知识’**（您的科研直觉、批判性思维和独特的分析框架）。”
  >
  > “请依次完成以下三个校准模块，来构建一个**既掌握您的知识、又擅长您的思考方式**的AI伙伴。”

- **核心视觉：“分身进化度”**

  - 界面的视觉焦点应是一个动态的“**分身进化度**”环形进度条。
  - 它必须清晰地显示一个加权百分比（例如：0%, 40%, 70%, 100%），游戏化地展示“数字分身”的构建进度。

- **核心激励：“80% 门槛”**

  - 这是我们激励设计的核心，也是解决“用户多样性”挑战的关键。我们必须在仪表盘上清晰地展示“功能门槛”。
  - **设计原理：** 这是一个“刚需”激励，它完美地**分层**了用户。90%的“任务型”用户可以只完成“模块一”（获得约40%进化度）并立即享用V1.0的价值（个性化RAG）。10%的“探究型”专家（我们的核心用户）则会被“解锁高级功能”所激励，从而**主动**完成“模块二”和“模块三”。
  - **UED 状态设计：**
    - **进化度 < 80% (锁定状态):**
      - `🔒 <文字>` **群体智能讨论班 (未激活)**
      - `🔒 <文字>` **分身能力分享 (未激活)**
      - **引导文案 (灰色小字):** “（‘分身进化度’达到 **80%** 后自动解锁）”
    - **进化度 >= 80% (解锁状态):**
      - `🔓 <文字>` **群体智能讨论班 (已激活)**
      - `🔓 <文字>` **分身能力分享 (已激活)**

### 2.2 奖励：客观的“认知画像”

这是对用户完成“校准”行为的即时奖励，也是我们获取用户反馈、完成AI对齐的关键闭环。

- **触发机制：** 当用户点击“分身进化度”的进度环时，应弹出一个详细的“认知画像”卡片。
- **设计要求 (给UED)：** **必须摒弃一切“标签”**。我们不做“算命”，我们只做客观的数据呈现。设计必须简洁、专业、富有设计感，体现“洞察”而非“评判”。
- **画像内容：**
  1. **“知识结构” (Knowledge Structure):**
     - **数据来源：** 模块一（RAG）。
     - **UED 呈现：** 可视化的动态词云或知识星系图。
     - *（示例）：“您的知识核心主要由以下节点构成：**纳米医学 (40%)**, **免疫学 (30%)**, **DNA纳米技术 (15%)**...”*
  2. **“思维洞察” (Thinking Insight):**
     - **数据来源：** 模块二（快速校准）和模块三（陪聊）。
     - **UED 呈现：** 简洁的雷达图或条形图。
     - *（示例）：“您的探究模式倾向于：”*
       - `<文字>` **机制深潜** (探究“如何运作”)
       - `<文字>` **系统关联** (探究“为何如此”与“矛盾点”)
       - `<文字>` **转化应用** (探究“有何价值”)
       - `<文字>` **概念定义** (探究“是什么”)
- **反馈闭环 (给Dev)：**
  - 画像卡片的底部**必须**包含“**认可此画像**”或“**修正我的画像**”的按钮。
  - 用户的“修正”（例如，用户认为自己的“转化应用”倾向应该更高）是**最高权重的AI对齐（Alignment）数据**，开发团队必须捕捉此反馈，用以迭代优化“数字分身”的模型。

## 3. 三个校准模块（给 UED & Dev 的具体任务）

这是用户“通关”以填充“进化度”的路径。UED团队在设计这三个卡片时，应使其在视觉上保持一致性，清晰地传达**价值**、**所需时间**和**完成状态**。

### 3.1 模块一：构建“知识核心”（显性知识）

- **UED 标题：** `模块一：构建“知识核心” (显性知识)`
- **UED 描述：** “上传您撰写或高度认可的文献。**您的‘数字分身 v1.0’将立即激活**，并作为‘无忧知识助手’的核心数据源，为您提供一个熟悉您知识背景的对话伙伴。”
- **即时价值 (V1.0)：** 这是我们为**所有用户**提供的“见面礼”。它解决了“精英悖论”，让90%的“任务型”用户获得即时满足。
- **进度奖励：** `（分身进化度 +40%）`
- **UED 状态：**
  - `[未完成]` 动作按钮：`[ 上传文献库 ]`
  - `[已完成]` 状态显示：`✅ 已激活 v1.0`
- **Dev 任务：** 构建稳定、高效的RAG（文献库） ingestion 管道。确保此模块的个性化RAG能被“无忧知识助手”立即调用。

### 3.2 模块二：校准“思维框架”（隐性知识 - 广度）

- **UED 标题：** `模块二：校准“思维框架” (隐性知识)`
- **UED 描述：** “用5分钟完成一次‘思维倾向评估’。我将向您展示几段文献摘要，请从4-5个选项中选出‘最像您会问的第一个问题’的那个选项。”
- **设计（V4方案）：** 这是一个低摩擦力的“多选题”测试。
  - **(给UED)** 选项**必须是“无标签”的**（例如，(A) "这个‘AND门控’的物理原理是什么？" 而**不是** (A) "一个机制(M)类问题"）。详见第4.1节。
- **进度奖励：** `（分身进化度 +30%）`
- **UED 状态：**
  - `[未完成]` 动作按钮：`[ 开始快速校准 (约5分钟) ]`
  - `[已完成]` 状态显示：`✅ 框架已校准`
- **Dev 任务：**
  1. 设计“探针库”（即用于测试的“摘要+选项”）。
  2. 构建“后台映射”逻辑：能将用户点击的“无标签选项”（如A/B/C/D）**准确映射**到我们内部的思维框架（如“机制/M”、“矛盾/S”、“转化/T”）。

### 3.3 模块三：锤炼“思维深度”（隐性知识 - 深度）

- **UED 标题：** `模块三：锤炼“思维深度” (隐性知识)`

- **UED 描述：** “与我进行一次‘苏格拉底式陪聊’。您可以提供一篇文献或话题，我将与您逐段共读，并在关键节点，**基于您的‘知识核心’（模块一）和‘思维框架’（模块二）**，向您发起‘启发式’讨论。”

- **设计（V6方案）：** 这是最高价值的“隐性知识”捕捉模块。

  1. **(给UED) 陪聊过程：** Agent **100%被动**，只回答，不反问，实现**零摩擦力**心流。

  2. **(给UED) 即时奖励（在用户点击“完成”时）：** **必须**弹窗给予即时奖励：“感谢您的探讨！您的分身刚刚基于您的X个深刻见解完成了进化！

     $$进度条 70% -> 100%$$

     $$高级功能已解锁！$$

     ”。详见第4.2节。

- **进度奖励：** `（分身进化度 +30%）`

- **UED 状态：**

  - `[未完成]` 动作按钮：`[ 开始一次“陪聊” ]`
  - `[已完成]` 状态显示：`✅ 深度已锤炼`

- **Dev 任务（核心）：**

  1. 构建一个“**被动式认知指纹分析器**”。
  2. 在“陪聊”过程中，该分析器必须能在**后台**、**实时**地将用户的**自由提问**（“鲜活思考”）准确分类到“机制/M”、“矛盾/S”等框架中，并将其存入“认知指纹”数据库。

### 4.1 模块二：“快速校准”交互样例 (V4方案)

- **(A)** “PGN”和“AND门控”的**具体定义**是什么？
- **(B)** 这种“AND门控”在物理化学上的**工作原理**是什么？它是如何做到“同时”需要两个信号才释放的？
- **(C)** 这个技术的**临床转化前景**如何？例如它的生产成本和体内（in vivo）的稳定性？

**(用户点击 (B))**

- *内部映射：(B) -> 归类于“机制深潜 (M)”维度。*
- *“机制”权重 +3。*

**探针 2 / 5 (关于潜在矛盾)**

**(用户点击 (B))**

**智能体 (Dev后台逻辑):**

### 4.2 模块三：“苏格拉底式陪聊”交互样例 (V6方案)
